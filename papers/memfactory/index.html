MemFactory 交互式论文解读

MemFactory: Unified Inference & Training Framework for Agent Memory 不是提出一个新的记忆算法，而是把 Memory-RL 研究里的 extraction、update、retrieval、agent、environment、trainer 统一成可插拔框架。它更像 LLaMA-Factory 之于 fine-tuning：降低搭建和比较记忆代理的工程门槛。

1. 核心判断

MemFactory 的价值在基础设施层：当 Memory-R1、RMM、MemAgent 这类方法都需要训练记忆策略时，研究者需要统一的模块接口、环境奖励、rollout 和评测管线，而不是为每篇论文重写一套实验系统。

一句话概括：MemFactory 把 memory lifecycle 拆成 Lego-like 组件，让 extraction / update / retrieval / agent policy 能在同一训练和推理框架里组合、替换和评测。

4. 实验结果：验证框架能复现并优化记忆策略

设置论文结果解读
验证对象使用公开 MemAgent 架构与训练/评测数据。说明 MemFactory 不是只停留在抽象接口，而是跑通了一个复杂 Memory-RL 范式。
Qwen3-1.7B平均分相对提升 14.8%，OOD benchmark 略降。小模型主任务收益明显，但迁移稳定性仍有限。
Qwen3-4B-Instruct平均相对提升 7.3%，包括 OOD benchmark 的一致提升。更大模型能更稳地迁移 recurrent memory policy。

5. 边界与工程启发

适合的场景

要比较多种 Memory-RL 策略、复现实验、替换 extractor/updater/retriever 的研究与工程团队。

不是完整产品系统

框架降低训练和评测门槛，但不等于已经提供用户权限、可视化编辑、数据治理和线上运维。
